# ADR-002. LangGraph 선택 (Agent 오케스트레이션)

- **Status**: Accepted
- **Date**: 2026-04-26

## 결정

Multi-Agent 오케스트레이션 프레임워크로 LangGraph 채택.

## 이유

1. **상태 관리 내장** — Write Path는 글감 선택 → 초안 생성 → 다듬기 → 플랫폼별 변환의 다단계 상태가 흐름. LangGraph는 그래프 노드 사이의 상태 전달을 1급으로 다룸.
2. **분기·반복 지원** — "초안 품질이 낮으면 다시 다듬기" 같은 조건부 분기, 반복 루프가 자연스러움.
3. **LangChain 생태계 호환** — RAG·도구 호출·LLM 추상화 등 기존 LangChain 자산 재사용 가능.

## 대안

- **단순 LangChain 체인** 탈락 — 분기 처리·상태 관리가 어려움. 직선적 파이프라인에는 적합하지만 Multi-Agent 워크플로우에 부적합.
- **CrewAI** 검토 — Agent 역할 추상화는 깔끔하지만 상태 흐름 제어가 LangGraph 대비 약함. 14일 일정에서 디버깅 가시성 면에서 LangGraph 우위.
- **자체 구현** 탈락 — 14일에서 오버스펙. 검증된 프레임워크 활용이 합리.

## 결과 / Trade-off

- (+) 다단계 워크플로우 자연스러움, 상태·분기·반복 무료, 생태계 풍부
- (−) 학습 곡선 있음 (그래프 사고방식)
- (−) LangChain 의존성 따라옴 (버전 변경 영향 받음)
