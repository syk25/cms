# ADR-005. uv 선택 (패키지 매니저)

- **Status**: Accepted
- **Date**: 2026-04-27

## 결정

패키지 매니저로 uv 채택 (pip + venv + pyenv 조합 대신).

## 이유

1. pip 대비 10~100배 빠름 — 의존성이 자주 바뀌는 14일 프로젝트에서 누적 시간 절약.
2. Python 버전 관리 + venv + lockfile 통합 — pyenv·venv 별도 도구 불필요.
3. PEP 621 표준(pyproject.toml) 준수 — uv.lock으로 재현 가능한 빌드(reproducible build) 확보.

## 대안

- **pip + venv + pyenv** 탈락 — 도구 분산, 속도 느림.
- **Poetry** 탈락 — 속도 측면에서 uv 우위, lockfile 형식이 비표준.

## 결과 / Trade-off

- (+) 셋업 속도, 의존성 갱신 속도, 단일 도구 일관성
- (−) 출시 1년차로 생태계 안정성 검증 진행 중, 사내 표준이 다른 환경에선 도입 마찰 가능
